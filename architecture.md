## Architecture
Users access WordPress via the EC2 public IP.
The EC2 instance hosts Docker, which runs WordPress and MySQL container.
WordPress communicates with the MySQL container over a Docker network, I used the default network. 
The MySQL data is stored on an attached EBS volume using a bind mount, ensuring persistence beyond container lifecycle. 
Backups are periodically taken and stored in S3.

Why is an EBS volume used for MySQL data instead of letting the container store it internally? What would happen without it?
The EBS volume is an extra storage layer on the EC2 instance where the data from mysql is written to, this is because if the mysql container crashes, the data is safe on the EBS volume. So data is not stored on the mysql container rather it is stored on the EBS volume attached to the EC2 instance.

If an EBS volume is not attached, whenever the container stops running the data is lost this is because the data lives inside the container. A new container means new data, but with the EBS volume, the data generated by any running mysql db will be stored on the volume and it won't be lost even when the container stops and starts as a new container. This is important for databases so that data is not lost.

My security group has port 80 and 22 open, the port 22 is to allow me access the server remotely and the port 80 is so that wordpress can be accessible to users from anywehre. Port 80 is not secured, a more secured port to expose my website should have been on the port 443 (hhtps). 
Port 22 is open, this is also not secured because it is receiving traffic from the web (0.0.0.0). 
My port 22 is secured, becuase it can only accept ssh from my laptop server


What would break if your EC2 instance crashed right now? What data would survive and what would be lost?
If my Ec2 instance crashed, wordpress will not be accessible and the ebs volume will also be not be lost since it is attached to the instance, if the instance crashed, the ebs volume will get detached and it can be reattached to another instance just like a usb. Also my mysql data is backed up to an s3 bucket which is outside of ec2 instance. Whatever data I have there is safe and can be restored.
